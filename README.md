# 🌊 N84 Employment Platform — Актау

**N84** — это интеллектуальная платформа для трудоустройства молодежи г. Актау, которая объединяет локальный малый бизнес и соискателей с помощью искусственного интеллекта.

---

## 🚀 Основная Концепция
Проект решает проблему разрыва между молодежью (ищущей подработку/работу) и малым бизнесом города. Вместо сложной бюрократии — быстрый вход через Telegram-бота и мгновенный "умный" поиск.

### Ключевые возможности:
*   **🤖 AI Matching (Qwen + pgvector)**: Мы используем векторную память для семантического поиска. Система не просто ищет слова, она понимает "смысл" опыта соискателя и требований бизнеса.
*   **⚡ Мгновенный отклик**: Работодатель получает уведомление в Telegram с ИИ-анализом кандидата и кнопкой для прямой связи.
*   **🔑 Гибридная Авторизация**: Регистрация через бота с мгновенной выдачей OTP-кода для входа на веб-панель.
*   **📍 Локальный фокус**: Учет микрорайонов Актау и поддержка национальной валюты (**₸**) во всех интерфейсах.
*   **🛡️ Надежность данных**: Реализовано каскадное удаление данных (PostgreSQL Cascade) для корректного управления вакансиями и откликами.

---

## 🛠 Технологический Стек
*   **Frontend/Backend**: Next.js 15 (App Router, TypeScript)
*   **Database**: Supabase (PostgreSQL) + **pgvector** (для AI-поиска)
*   **Bot Framework**: grammY
*   **AI Engine**: Qwen 2.5 (32B/72B) via Dashscope
*   **Hosting**: Vercel

---

## 🧪 Система Тестирования и Контроля
Мы внедрили автоматизированную систему проверки здоровья проекта, чтобы гарантировать стабильность перед жюри.

**Комплексный тест:** `tests/health_check.js`

### Что проверяют тесты:
1.  **AI Connectivity**: Реальный запрос к Qwen с проверкой формата ответа.
2.  **Vector Memory**: Проверка доступности расширения `vector` в БД.
3.  **Telegram Webhook**: Диагностика связи между серверами Telegram и Vercel.
4.  **Database Integrity**: Проверка наличия всех 20+ критических колонок (БИН, Район, О себе и т.д.).
5.  **Route Availability**: Пинг жизненно важных API-эндпоинтов для предотвращения 404/500 ошибок.

**Как запустить тесты:**
```bash
node tests/health_check.js
```

## ⚙️ Установка и Настройка
1. Скопируйте `.env.example` в `.env.local`.
2. Заполните ключи:
   * `TELEGRAM_BOT_TOKEN`
   * `SUPABASE_SERVICE_ROLE_KEY`
   * `QWEN_API_KEY`
   * `QWEN_MODEL` (например: `qwen-max` или `qwen2.5-32b-instruct`)
3. Запустите миграцию для подготовки базы:
   ```bash
   node src/scripts/mega_fix.js
   ```
4. Установите зависимости и запустите проект:
   ```bash
   npm install
   npm run dev
   ```
