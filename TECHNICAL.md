# 🛠 Техническая Документация N84

Проект N84 построен на современном стеке технологий с фокусом на масштабируемость и использование ИИ в каждой части системы.

## 🏗 Архитектура Системы

Система состоит из четырех ключевых слоев:

1.  **Frontend**: Next.js (App Router) с адаптивным дизайном, оптимизированным для мобильных устройств (Mobile-First).
2.  **API Layer**: Серверные роуты (Edge/Node.js) для обработки бизнес-логики и интеграции с Telegram/AI.
3.  **Database**: Supabase (PostgreSQL) с расширением **pgvector** для хранения семантических смыслов вакансий.
4.  **AI Orchestrator**: Интеграция с моделями **Qwen (Alibaba Cloud)** для интеллектуального поиска и анализа.

---

## 🤖 Интеллектуальные Функции (AI Core)

### 1. Семантический матчинг (Matching Engine)
В отличие от классического поиска по ключевым словам, N84 использует LLM для глубокого анализа.
*   **Логика**: При создании вакансии система анализирует её описание, требования и локацию.
*   **Процесс**: ИИ сравнивает эти данные с профилями соискателей в БД и выставляет оценку совместимости (Match Score) от 0 до 10.
*   **Результат**: Только релевантные кандидаты (Score >= 5/10) получают уведомление в Telegram.

### 2. ИИ-анализ для работодателей
При каждом отклике работодатель получает не просто имя, а сгенерированный ИИ вердикт: "Почему этот кандидат подходит?". Это на 60% ускоряет первичный отбор кадров.

---

## 📱 Интеграция с Telegram

*   **Telegram WebApp**: Позволяет соискателям просматривать вакансии и откликаться прямо внутри Telegram, не выходя из мессенджера.
*   **Мгновенные уведомления**: Двусторонняя связь между работодателем и соискателем через Bot API.
*   **OTP-авторизация**: Безопасный и быстрый вход на сайт через проверочный код в боте.

---

## 🧪 Надежность и Тестирование

Проект включает в себя встроенную систему диагностики:
*   `tests/health_check.js`: Полная проверка API, БД, ИИ и Webhook-статуса.
*   `scripts/send_test_push.js`: Инструмент для проверки сквозной доставки уведомлений.
*   `scripts/init_vector.js`: Скрипт инициализации векторной памяти в PostgreSQL.

---

## 🛡️ Целостность Данных
В архитектуры БД заложены принципы **ON DELETE CASCADE**, что гарантирует чистоту данных: при удалении вакансии система автоматически очищает все связанные отклики и уведомления.
